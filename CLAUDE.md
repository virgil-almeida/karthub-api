## Project Overview

KartHub (branded as **KartClub**) is a social platform for kart racing pilots. It is a monorepo with a single frontend application — the backend is Supabase (PostgreSQL + Auth + Storage + Edge Functions). There is no custom backend server; all data access goes through the Supabase client.

## Frontend Commands

All commands run from `frontend/`:

```bash
cd frontend

npm run dev          # Start dev server on port 8080
npm run build        # Production build
npm run lint         # ESLint
npm run test         # Run unit tests once (Vitest + jsdom)
npm run test:watch   # Run tests in watch mode
npx playwright test  # Run e2e tests
```

Run a single test file:
```bash
cd frontend && npx vitest run src/path/to/file.test.tsx
```

## Environment Variables

`frontend/.env` must have:
```
VITE_SUPABASE_URL=
VITE_SUPABASE_PUBLISHABLE_KEY=
VITE_SUPABASE_PROJECT_ID=
```

## Architecture

### Tech Stack
- **React 18** with TypeScript, Vite, React Router v6
- **Supabase** (`@supabase/supabase-js`) — sole backend; client at `src/integrations/supabase/client.ts`
- **TanStack Query** — all server state, data fetching, and mutations
- **shadcn/ui** (Radix UI + Tailwind CSS) — UI component library
- **react-hook-form + zod** — forms and validation
- **i18next** — internationalization (pt-BR default, en, es)

### Data Flow
All data queries go through custom hooks in `src/hooks/`. Each hook uses `@tanstack/react-query` (`useQuery`/`useMutation`) to call Supabase directly. The generated TypeScript types from the database schema live in `src/integrations/supabase/types.ts` — **do not edit this file manually**.

### Auth & Permissions
`src/contexts/AuthContext.tsx` wraps the app and exposes `useAuth()`. Permissions are determined via Supabase RPC calls (`get_user_tier`, `is_admin`, `can_view_analytics`, `can_create_championships`). Tiers: `free → user → plus → moderator → admin`.

### Key Domain Entities
Defined in `src/types/kart.ts`:
- **Championship** → **Events** (at a Track) → **Heats** → **HeatResults** → **LapTelemetry**
- **Profiles** (pilots) with **DriverBadges** and subscription **tiers** (`user_roles` table)

### Error Handling
Use `createUserFriendlyError()` from `src/lib/errorHandler.ts` when throwing errors from hooks. It maps Supabase/PostgreSQL error codes to Portuguese user messages and strips internal details.

### Internationalization
All user-visible strings must use `useTranslation()` from `react-i18next`. Translation keys live in `src/i18n/locales/{pt-BR,en,es}.json`. Always add keys to all three locale files.

### Database Migrations
Supabase migrations are in `frontend/supabase/migrations/`. Apply with the Supabase CLI (`supabase db push`). The `supabase/config.toml` targets the remote project.

### Path Alias
`@/` maps to `src/`. Use it for all imports within the project.
